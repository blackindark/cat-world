# 电商平台专业领域知识清单

这个仓库现在要朝“前台 + 中后台 + 技术架构 + 产品架构”一体化的电商学习平台演进。

## 一、前台业务域
- 首页/频道/会场
- 搜索、筛选、排序、推荐
- 商品详情页（SPU/SKU、价格、库存、活动、评价、问答）
- 购物车、结算、下单
- 支付、支付回调、风控校验
- 订单查询、物流轨迹、签收确认
- 会员中心、优惠券、积分、余额、礼品卡
- 售后申请：退款、退货、换货、投诉
- 内容种草、直播导购、私域导购、消息订阅

## 二、中后台业务域
- 商品中心：类目、属性、品牌、SPU/SKU、上下架、价格、库存
- 会员中心：用户画像、生命周期、分层、标签、权益
- 交易中心：购物车、订单、支付、发票、对账
- 营销中心：促销、优惠券、满减、会员价、拼团、秒杀、预售、直播活动
- 履约中心：OMS、WMS、TMS、门店自提、逆向物流
- 客服售后：工单、退款审核、质检、赔付、投诉
- 财务结算：收入确认、退款冲销、渠道结算、商家结算、税务票据
- 风控合规：刷单、黄牛、欺诈、账号风险、支付风控、隐私合规
- 内容与 CMS：活动页、频道页、导购内容、UGC 审核

## 三、产品架构学习重点
- 统一商品模型：SPU/SKU/类目/属性/价格/库存
- 统一订单模型：下单、支付、拆单、发货、售后、逆向单
- 统一营销模型：活动规则引擎、价格叠加优先级、用户分层命中
- 统一会员模型：积分、成长值、权益包、标签体系
- 统一履约模型：仓配模式、区域库存、时效承诺、逆向流程
- 统一数据模型：GMV、转化率、复购率、退款率、履约时效、广告 ROI

## 四、技术架构学习重点
- 前台：SSR/ISR、推荐、搜索、埋点、性能、A/B、BFF
- 中台：商品、库存、订单、营销、会员、支付、售后拆分
- 异步一致性：库存预扣、支付确认、订单状态推进、退款补偿
- 风控：下单限流、黑名单、地址风险、设备指纹、支付风控
- 数据：ODS -> DWD -> DWS -> ADS，实时与离线结合
- 可观测性：订单漏单、库存超卖、支付丢单、履约 SLA、退款时长

## 五、建议实现优先级
1. 商品中心
2. 客户中心
3. 交易中心
4. 营销中心
5. 售后中心
6. 数据架构与指标看板
